In order to better understand the conversion between bases, start with decimal (base-10) which most are familiar with. All numbers represented in decimal consist of a string of digits, each corresponding to a decimal place and consisting of the 10 digits 0-9. consider the number 123, the value of which can be broken down in the following way:


123=100+20+3

100=10^2*1

20=10^1*2

3=10^0*3


While this may seem obvious when representing a decimal value it is a helpful framework for understanding other bases and how to convert between them. Base-10 uses 10 digits (0-9) and each position in the string represents that digit value multiplied by the base raised to said position. Other base number representations work in the same way. Consider the binary (base-2) number 101. The value can be determined in the same way as with decimal numbers:


5=4+0+1

4=2^2*1

0=2^1*0

1=2^0*1


The value of 101 can be calculated as 5 by taking each digit, multiplying it by 2 raised to the positional power, and taking the sum. Binary numbers are represented using only 2 digits (0, 1) and each positional value has a base of 2, rather than 10 in decimal.


Hexadecimal (base-16) continues the pattern. Because the arabic numeral symbols only go up to 9, 10-15 are represented as the letters a-f. Therefore the value of the hexadecimal number f13 is 3859:


3859=3840+16+3

3840=16^2*f

16=16^1*1

3=16^0*3
